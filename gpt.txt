Great question — you're describing a classic layout pattern where:

The footer is always visible (sticky/fixed) at the bottom

The main content area scrolls independently

Layout proportions (e.g. `% heights) are based only on the scrollable "body" area — not including the footer height

✅ Here’s how to achieve that, cleanly:
🧱 HTML Layout
html

Editar
<body>
  <div class="page-wrapper">
    <header>Header (optional)</header>

    <div class="body-content">
      <main>
        <!-- Your scrollable content here -->
      </main>
    </div>

    <footer>
      Footer stays here always
    </footer>
  </div>
</body>
🎯 CSS Strategy
css
Copiar
Editar
html, body {
  height: 100%;
  margin: 0;
}

.page-wrapper {
  display: flex;
  flex-direction: column;
  height: 100vh; /* total viewport height */
}

footer {
  height: 80px; /* or whatever height you want */
  flex-shrink: 0;
  background: #eee;
  text-align: center;
  line-height: 80px;
  border-top: 1px solid #ccc;
  position: relative;
  z-index: 1;
}

.body-content {
  flex: 1; /* take up remaining space */
  overflow-y: auto; /* only this part scrolls */
  position: relative;
}
🔍 What this does:
The .page-wrapper is a full-height column layout (100vh)

The footer has a fixed height and doesn’t shrink

The .body-content expands to fill all space except the footer

Only .body-content scrolls, not the entire page

You can now use % inside .body-content, and it will not include the footer

🧪 Example height usage inside .body-content:
css
Copiar
Editar
.left-container {
  height: 20%; /* 20% of .body-content, NOT including footer */
}
🚨 Gotchas
Don't use position: fixed for the footer in this pattern (you don't need it and it breaks percentage-based layouts).

overflow-y: auto must be set only on the scrolling section.
